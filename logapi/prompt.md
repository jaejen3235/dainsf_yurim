# [MES 로그 수집 API 연동 및 시뮬레이터 개발 요구사항]

## 1. 역할 및 목표
당신은 PHP 기반의 MES 솔루션 백엔드 개발 전문가이자 서버 관리자입니다. 정부(스마트공장 사업관리시스템)의 지침에 따라 MES 사용 파악용 로그를 전송하는 '로그 수집 API 연동' 코드를 파이선과 같이 독립 실행 가능한 언어를 사용해서 작성해야 합니다. 또한, 이를 자동화하기 위해 DB 저장 없이 지정된 디렉토리의 파일에 로그를 기록하고, `logrotate`를 통해 백업을 관리하는 **'로그 API 호출 시뮬레이터'** 스크립트를 작성해야 합니다.

## 2. API 연동 기본 정보
- **전송 방식**: POST 방식 전용.
- **서비스 URL**: `https://log.smart-factory.kr/apisvc/sendLogData.json`.
- **통신 데이터 타입**: JSON 객체 전송.
- **응답 검증**: 정상 전송 시 수신 결과 코드(`recptnRsltCd`) 값으로 **'AP1002'**가 반환되므로, 이를 기준으로 성공/실패 여부를 판별하세요.

## 3. 입력 데이터 구조 및 제약 사항
아래 6개의 파라미터를 포함하는 JSON 객체를 구성해야 합니다.
1. `crtfcKey` (로그API인증키): 외부에서 주입받는 인증키 (String, 최대 100자).
2. `logDt` (로그일시): `YYYY-MM-DD HH:MI:SS.SSS` 형식으로 24시간 기준 밀리초까지 표현하며 당일 로그만 전송 가능합니다.
3. `useSe` (접속구분): 사용자의 액션에 따라 아래 코드로 변환하여 전송합니다.
   - 로그인: DO6001
   - 로그아웃: DO6002
   - 데이터 조회(R): DO6003
   - 데이터 등록(C): DO6004
   - 데이터 수정(U): DO6005
   - 데이터 삭제(D): DO6006
   - 입출고 시간: DO6007
   - 테스트: DO6999
4. `sysUser` (사용자): 현재 로그인 중인 사용자의 ID, 이름, 사번 중 하나를 전송합니다.
5. `conectIp` (IP정보): Frontend가 실행되는 PC의 IP 주소입니다.
6. `dataUsgqty` (데이터사용량): API Payload의 byte 수량이며, 값이 없으면 '0'으로 전송합니다.

*주의사항*: 전송된 로그 데이터는 최소 10분의 간격을 두고 저장되어야 하며, 하루 최대 5,000건 전송 제한(초과 시 AP1031 에러)이 있습니다.

## 4. 시뮬레이터 동작 로직 (PHP 스크립트)
Crontab 등을 통해 실행될 PHP 시뮬레이터를 다음 시나리오에 맞춰 작성하세요.
- **작업 시간**: 업무시간인 09:20 ~ 17:30 사이클로 동작합니다.
- **점심시간 처리**: 점심시간대에는 가급적 로그 API 호출 동작이 발생하지 않도록 제어합니다.
- **오전/오후 세션 접속 로직**:
  - 오전 시간대 및 오후 시간대에 사전에 정의된 사용자 목록(Array)에서 무작위로 1명 이상이 로그인(DO6001) 액션을 발생시킵니다.
  - 로그인 후 무작위로 지정된 시간(1시간 ~ 2시간) 동안 접속을 유지한 뒤 로그아웃(DO6002) 액션을 발생시킵니다.
- **세션 중 액션 발생 로직**:
  - 접속이 유지되는 시간 동안 **15분 ~ 20분 사이의 랜덤한 주기**로 조회, 등록, 수정, 삭제(DO6003~DO6006) 중 1개 이상의 액션 로그 API를 호출합니다 (이 주기는 최소 10분 이상 간격 조건을 충족합니다).
- **데이터 사용량**:
  - 로그인, 로그아웃은 0바이트로
  - 조회는 수십kb 랜덤 생성
  - 등록, 수정, 삭제는 수kb 랜덤 생성

## 5. 결과 기록 (로컬 파일 로깅)
- 별도의 DB 테이블에 로그를 적재하지 않습니다.
- 시뮬레이터가 API를 호출할 때마다 지정된 폴더에 있는 `logapi.log` 파일에 로그를 텍스트 형태로 Append(추가) 하는 함수를 작성하세요.
- 기록할 내용은 실행 일시, 요청 Payload 데이터, 응답 수신코드 및 상세 결과입니다.

## 6. 로그 파일 백업 관리 (logrotate 설정)
- 파일 시스템 용량 관리를 위해 Linux의 `logrotate`를 활용합니다.
- 작성된 `logapi.log` 파일이 **최대 5MB 크기에 도달하면 로테이트** 시키고, **최대 5개의 백업 파일만 유지**하도록 `/etc/logrotate.d/` 디렉토리에 들어갈 설정 파일 스크립트를 작성해 주세요.
